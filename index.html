Modern Architecture

Built on a foundation of 18 independent microservices with a unified API Gateway. Each service owns its data, scales independently, and communicates through well-defined contracts. Our schema-per-service PostgreSQL architecture ensures data isolation while enabling cross-service queries when needed.

4-Layer Security Model

Defense-in-depth security with authorization at every layer. Each layer adds protection, ensuring that even if one layer is bypassed, others maintain security.

L1
Frontend (UI Visibility)
usePermissions() hook for cached permission checks. Show/hide UI elements based on role + module status. UX optimization only.
React Hooks UI Filtering Fail-Open
L2
API Gateway (Authentication)
JWT validation and token refresh. Basic role verification. Rate limiting and request validation. Returns 401/403.
JWT Rate Limiting CORS
L3
Microservice (Business Logic)
Detailed permission checks with organizational scope. Field-level access control. Rich error messages. Business context awareness.
Org Scope Field-Level Business Rules
L4
PostgreSQL RLS (Final Safety Net)
Tenant isolation mandatory on all tables. Basic role verification as fail-safe. Simple, fast policies. Query returns no rows on failure.
Tenant Isolation RLS Policies Fail-Safe

Project Organization

The backend consists of 18 microservices with shared libraries for types, utilities, and event bus. Each service follows a consistent structure with controllers, services, repositories, and routes.
